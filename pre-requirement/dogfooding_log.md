# Dogfooding Log

이 하네스는 자기 자신을 사용해서(dogfooding) 개발되었다.
2회의 dogfooding을 통해 하네스 프로세스를 검증하고, 발견된 문제점을 즉시 반영했다.

---

## 1회차 — 하네스 자체를 킥오프

대상: AI Project Kickoff Harness (이 프로젝트 자체)

### 실행 흐름

| 단계 | 결과 |
|---|---|
| 1. 아이디어 입력 | "아이디어를 구체화하고 기획하고 오류를 잡는 하네스" |
| 2. 유형 판별 | CLI 도구 / 자동화 오케스트레이터 |
| 3. 인터뷰 | 7개 질문 (입력방식, 산출물, 저장위치, 스킬세팅 등) |
| 3.5 AI 제안 | 1회차에서는 미적용 (이 단계 자체가 2회차에서 추가됨) |
| 4. 프로필 생성 | project_profile.md 생성 |
| 5. 누락/모순 점검 | 모순 2건, 누락 4건 발견 및 해결 |
| 8. 체크리스트 | dev_checklist.md 생성 |
| 9. 스킬 세팅 | 기본 7개 스킬 복사 완료 |

### 발견 및 반영 사항

| 발견 | 반영 |
|---|---|
| Plan.txt에서 산출물 포맷이 JSON으로 되어 있었으나 Markdown으로 확정됨 | Plan.txt 수정 |
| README.md가 MVP 산출물에 포함되어 있었으나 v2로 이동 | Plan.txt 수정 |
| 다른 CLI 지원 방식이 미정이었음 | Claude Code 전용으로 확정, 나머지 v2 |
| 하네스 구조가 미정이었음 | 단계별 분리 스킬로 확정 |

## 2회차 — 다른 프로젝트에 하네스 적용

대상: 한국 주식 + 거시경제 통합 분석 포트폴리오

### 실행 흐름

| 단계 | 결과 |
|---|---|
| 1. 아이디어 입력 | "한국 주식 + 거시경제 통합 분석, 실무 구조 구현, Power BI, AI 인사이트" |
| 2. 유형 판별 | 데이터 파이프라인 + BI 대시보드 + AI (복합) |
| 3. 인터뷰 | B+C 방식 적용 (묶음 질문 + 부족 시 추가). 기술 질문에 AI 추천 제공 |
| 3.5 AI 제안 | 5개 제안 → 2개 채택(데이터 품질 대시보드, 자동 리포트), 1개 v2(인사이트 히스토리) |
| 4. 프로필 생성 | 구두 확인 (파일 생성 생략) |
| 5. 누락/모순 점검 | 모순 1건, 누락 4건 발견 및 해결 |
| 8. 체크리스트 | 구두 확인 |
| 9. 스킬 세팅 | 흐름 검증만 |

### 발견 및 반영 사항

| 발견 | 반영 |
|---|---|
| 기술적 질문에 사용자가 답 못할 수 있음 | 인터뷰 규칙 추가: "잘 모르겠어"/"추천해줘" 허용, What/How 분리 |
| AI 추천이 "쉬운 것"을 우선하면 안 됨 | 인터뷰 규칙 추가: "목적에 맞는 것"을 우선 추천 |
| 3.5단계에서 난이도/시간 정보가 있으면 채택 판단이 쉬움 | 3.5단계 규칙에 난이도+예상시간 제공 추가 |
| Step 3.5 자체가 유용함을 확인 | MVP에 3.5단계 확정 |

## 3회차 — 자동화된 하네스 통합 테스트 (Phase 7)

대상: HR 채용 데이터 분석 포트폴리오 (hr_data_analytics)

### 실행 흐름

| 단계 | 스킬 | 결과 |
|---|---|---|
| 1. 아이디어 입력 | kickoff-start | "HR 데이터 로드/가공 + AI 분석 + 시각화, datalake-warehouse-mart 구조" |
| 2. 유형 판별 | kickoff-start | 데이터 파이프라인 + AI 프로젝트 (부가: BI 대시보드) |
| 3. 기획 인터뷰 | kickoff-interview | 공통 7개 + 유형별 4개 질문. "추천해줘" 다수 → AI 추천 제공 |
| 3-2. 설계 인터뷰 | kickoff-interview | 4개 영역 완료 (파이프라인, 데이터구조, 볼륨, 실패시나리오) |
| 3.5 AI 제안 | kickoff-suggest | 5개 제안 → 전부 채택 (MVP 3개, 이후 확장 2개) |
| 4. 프로필 생성 | kickoff-profile | hr_data_analytics_kickoff.md 생성 (섹션 1~6) |
| 5. 누락/모순 점검 | kickoff-gap | 2건 발견 (누락 1, 모순 1) → 전체 해결 |
| 8. 체크리스트 | kickoff-checklist | 18개 항목 생성 |
| 9. 스킬 세팅 | kickoff-skills | 스킵 (이미 설치 완료 상태) |

### 발견 및 반영 사항

| 발견 | 심각도 | 반영 |
|---|---|---|
| kickoff-profile 실행 시 상태 파일에 섹션 5 추가 누락 | 낮 | kickoff_state.md에 섹션 5 수동 추가로 해결 |
| kickoff-gap에서 Google Sheets 역할 모순 감지 | 중 | 기술 스택에서 제거, API 직접 수집으로 문서 수정 |
| kickoff-gap에서 API 인증 관리 누락 감지 | 중 | .env 파일 관리 방식으로 제약사항에 추가 |

### 산출물

| 파일 | 상태 |
|---|---|
| output/hr_data_analytics/kickoff_state.md | ✅ 섹션 0~7 완성 |
| output/hr_data_analytics/hr_data_analytics_kickoff.md | ✅ 섹션 1~6 완성 |

## Dogfooding 종합 결론

- 3회 실행으로 총 8건의 설계 개선이 이루어짐
- 하네스 프로세스(1→2→3→3.5→4→5→8→9)가 끊김 없이 동작함을 확인
- 주요 개선: 인터뷰 규칙 3건 추가, 3.5단계 MVP 편입, 산출물 단일 파일화
- 3회차 추가 확인: 설계 인터뷰 포함 풀 코스 정상 동작, gap 점검이 실제 모순 감지 가능
