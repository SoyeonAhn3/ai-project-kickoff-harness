# 컨텍스트 질문 가이드

kickoff-context 스킬이 수집하는 6개 항목의 질문 분기 규칙.

---

## 질문 항목

| # | 항목 | 질문 | 선택지 | 자유 입력 |
|---|---|---|---|---|
| 1 | 프로젝트 목적 | 이 프로젝트의 주된 목적은? | 포트폴리오 / 업무 자동화 / 학습 / 실무 투입 | 허용 |
| 2 | 대상 독자/평가자 | 결과물을 누가 보는가? | 채용 담당자 / 팀장 / 교수 / 본인 | 허용 |
| 3 | 현재 기술 수준 | 기술 수준은 어느 정도? | 초급 / 중급 / 고급 | 구체 역량 추가 입력 권장 |
| 4 | 목표 직무 | 목표 직무가 있는가? | 자유 입력 | "없음" 허용 |
| 5 | 시간 예산 | 프로젝트에 쓸 수 있는 시간? | 1~2주 / 1개월 / 2~3개월 / 그 이상 | 허용 |
| 6 | 차별화 키워드 | 특히 보여주고 싶은 역량? | 자유 입력 | "없음" 허용 |

---

## 목적별 질문 톤 조정

### 포트폴리오

- 대상 독자 질문 강조 ("채용 담당자가 어떤 점을 볼까요?")
- 차별화 키워드 중요 — "이 프로젝트에서 뭘 보여주고 싶으세요?"

### 업무 자동화

- 시간 예산 질문 강조 ("현재 수동으로 하는 작업에 얼마나 시간이 걸리나요?")
- 대상 독자 → "이 결과를 보고받는 사람은 누구인가요?"

### 학습

- 기술 수준 질문 강조 ("이미 알고 있는 것과 배우고 싶은 것을 구분해주세요")
- 차별화 키워드 → "이 프로젝트를 통해 어떤 역량을 키우고 싶으세요?"

### 실무 투입

- 시간 예산 + 대상 독자 모두 중요
- 추가 질문 가능: "SLA나 성능 요구사항이 있나요?"

---

## 답변 처리 규칙

| 상황 | 처리 |
|---|---|
| 번호로 답변 | 해당 선택지 매핑 |
| 자유 텍스트 | 가장 가까운 선택지에 매핑 후 확인 |
| "잘 모르겠어" | 목적에 따른 기본값 제안 후 확인 |
| "패스" | 해당 항목 "미설정" 기록, 이후 스킬에서 기본 동작 |
| 복수 선택 | 주요 1개 확인 요청 |
