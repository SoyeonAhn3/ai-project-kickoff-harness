# kickoff-context Skill

"너는 누구고, 왜 이걸 만들어?"를 먼저 파악하여 이후 인터뷰, 제안, 평가, 체크리스트가 사용자 맥락에 맞게 동작하도록 하는 컨텍스트 수집 스킬.

---

## 사전 조건

- 없음 (킥오프 흐름의 첫 번째 스킬)
- 선택적 스킬 — 패스 시 이후 스킬이 context 없이도 정상 동작

---

## STEP 1 — 목적 확인

사용자에게 프로젝트의 주된 목적을 확인한다.

### 출력 형식

```
## 🎯 프로젝트 컨텍스트를 설정합니다

이후 기획/평가/체크리스트가 맞춤형으로 동작하기 위해 몇 가지 배경을 여쭤보겠습니다.

---

**1. 이 프로젝트의 주된 목적은 무엇인가요?**

1) 포트폴리오 — 취업/이직용 역량 증명
2) 업무 자동화 — 현재 수동 프로세스 개선
3) 학습 — 새로운 기술/개념 습득
4) 실무 투입 — 실제 서비스/시스템 구축

번호 또는 자유롭게 답변해주세요.
```

### 답변 처리

- 번호 선택 → 해당 목적 확정
- 자유 입력 → 4개 중 가장 가까운 것에 매핑 후 확인
- 복수 선택 → 주요 목적 1개 확인 ("둘 다 해당하면, 가장 중요한 하나를 골라주세요")

---

## STEP 2 — 수준/역량/시간 파악

목적 확정 후 기술 수준과 시간 예산을 파악한다.

### 출력 형식

```
**2. 현재 기술 수준은 어느 정도인가요?**

1) 초급 — 기본 문법/개념 학습 중
2) 중급 — 간단한 프로젝트 경험 있음
3) 고급 — 실무 프로젝트 다수 경험

구체적으로 다룰 줄 아는 기술이 있으면 함께 말씀해주세요.
(예: "중급, Python과 SQL은 할 줄 알아")

**3. 시간 예산은 어느 정도인가요?**

1) 1~2주
2) 1개월
3) 2~3개월
4) 그 이상
```

---

## STEP 3 — 목표/차별화 파악

`references/context-questions.md`를 참조하여 목적별 추가 질문을 한다.

### 출력 형식

```
**4. 대상 독자/평가자는 누구인가요?**

1) 채용 담당자
2) 팀장/매니저
3) 교수/멘토
4) 본인 (학습 기록용)

**5. 목표 직무가 있나요?**

자유 입력 (예: "데이터 엔지니어", "백엔드 개발자", "ML 엔지니어")
없으면 "없음"

**6. 이 프로젝트에서 특히 보여주고 싶은 역량이 있나요?**

자유 입력 (예: "ETL 설계 능력", "API 연동", "데이터 모델링")
없으면 "없음"
```

---

## STEP 4 — 확인 및 완료 안내

수집된 정보를 정리하여 사용자에게 확인을 받는다.

### 출력 형식

```
## ✅ 컨텍스트 수집 완료

| 항목 | 내용 |
|---|---|
| 프로젝트 목적 | [확정된 목적] |
| 대상 독자/평가자 | [확정된 대상] |
| 현재 기술 수준 | [수준 + 구체 역량] |
| 목표 직무 | [직무명 또는 "없음"] |
| 시간 예산 | [기간] |
| 차별화 방향 | [키워드 또는 "없음"] |

수정할 부분이 있으면 말씀해주세요.
없으면 `/kickoff-start`로 프로젝트 킥오프를 시작합니다.
```

### 데이터 전달 방식

이 스킬은 **파일을 생성하지 않는다**. 수집된 컨텍스트는 대화 컨텍스트로 전달된다.
`kickoff-start`가 상태 파일을 생성할 때 컨텍스트 데이터가 있으면 **섹션 0-1**로 포함한다.

---

## 출력 형식

이 스킬은 **파일을 생성/수정하지 않는다**:
- 수집된 데이터는 대화 컨텍스트에 존재
- `kickoff-start`가 상태 파일 생성 시 섹션 0-1로 영속화
- context 없이 start를 실행해도 정상 동작 (섹션 0-1 생략)

---

## 실패 처리

| 실패 유형 | 처리 방법 |
|---|---|
| 사용자가 전부 패스 요청 | "컨텍스트 없이 진행합니다" 안내 후 `/kickoff-start` 안내 |
| 답변이 선택지에 없음 | 사용자 입력을 그대로 기록 |
| 목적이 여러 개 | 주요 목적 1개를 선택하도록 유도 |

---

## 주의사항

- 질문은 총 6개 — 3턴 이내로 끝내야 함 (STEP 1: 1개, STEP 2: 2개, STEP 3: 3개)
- 선택지를 제시하되 자유 입력도 허용
- 수집한 정보는 이후 스킬에서 "강제"가 아닌 "가이드"로 사용됨
- 사용자가 "패스"하면 이후 스킬은 기본 동작으로 정상 진행
