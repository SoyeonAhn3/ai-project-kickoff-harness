# 기획 인터뷰 질문 가이드

## 공통 질문 (모든 유형)

아래 7개 질문은 유형에 관계없이 반드시 수집해야 할 핵심 정보다.

| # | 질문 | 수집 목적 |
|---|---|---|
| P1 | 이 프로젝트의 주요 사용자는 누구인가요? | 타겟 사용자 정의 |
| P2 | 이 프로젝트의 목적은 무엇인가요? (해결할 문제 또는 달성할 목표) | 핵심 목적 명확화 |
| P3 | 입력 데이터는 어디서 오나요? (파일, DB, API 등) | 데이터 소스 파악 |
| P4 | 결과는 어떤 형태로 보고 싶으신가요? (화면, 파일, 알림 등) | 산출물 형태 결정 |
| P5 | 이 프로젝트의 핵심 기능을 3~5개로 나열해주세요. | 기능 범위 파악 |
| P6 | 위 기능 중 MVP에 반드시 포함할 것은 무엇인가요? | 스코프 제한 |
| P7 | 제약사항이 있나요? (기한, 예산, 환경 제약 — 예: Docker 불가, Windows만, 사내망만 등) | 현실적 제약 반영 |

## 유형별 추가 질문

### AI 프로젝트

| # | 질문 | 수집 목적 |
|---|---|---|
| AI-1 | AI가 구체적으로 어떤 판단/예측을 해야 하나요? | AI 역할 범위 |

| AI-3 | AI 결과가 틀렸을 때 어떻게 처리하나요? (사람 검토, 자동 재시도 등) | 폴백 전략 |

## 질문 운용 규칙

- **1회차**: 공통 7개 + 유형별 3개 = 최대 10개를 한 묶음으로 제시
- **복합 유형**: 주요 유형의 추가 질문 3개 + 부가 유형에서 가장 중요한 1개 선택
- **추가 질문(C방식)**: 답변 후 정보가 부족한 영역이 있으면 1~2개 추가 질문
- **"잘 모르겠어" 답변 시**: 즉시 AI 추천을 제시 (목적에 맞는 것 우선)
- **사소한 디테일 금지**: 킥오프 수준 — 구현 세부사항은 묻지 않음
