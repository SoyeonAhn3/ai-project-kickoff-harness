# Test Scenario Generator Skill

개발 완료 시점에 맞는 사용자 테스트 시나리오를 생성하고 파일로 저장한다.
사용자가 테스트 완료 후 결과를 보고하면 시나리오 파일에 직접 기록한다.
생성 규칙은 `references/rules.md`, 파일 형식은 `references/template.md`를 참고한다.

---

## STEP 0 — 레퍼런스 로드

시나리오 생성 전 반드시 먼저 실행:

```
Read(".claude/skills/test-scenario/references/rules.md")
Read(".claude/skills/test-scenario/references/template.md")
```

→ 3축 구성 비율, 필수 항목, 파일 경로 규칙, 출력 포맷 확인

---

## STEP 1 — 입력 수집

사용자 설명에서 아래 3가지가 파악되면 바로 STEP 2 진행:
- 구현한 기능이 무엇인지
- 사용자가 하는 주요 동작
- 어디에 저장되거나 어떤 결과가 나오는지

파악이 부족하면 한 번만 질문:
```
"어떤 기능을 구현하셨나요? 간단히 알려주시면 바로 생성합니다:
1. 구현한 기능 요약
2. 사용자가 하는 주요 동작
3. 저장 위치 또는 결과물"
```

---

## STEP 2 — 시나리오 생성

`references/rules.md`의 3축 구성 비율과 필수 항목 기준으로 시나리오 작성.

---

## STEP 3 — 파일 저장

`references/template.md`의 파일 경로 규칙과 내용 구조에 맞게 저장.

1. `test-scenarios/` 폴더 없으면 생성: `mkdir -p test-scenarios`
2. `template.md`의 파일 내용 구조로 마크다운 작성
3. Write 도구로 저장

---

## STEP 4 — 채팅 출력

`references/template.md`의 채팅 출력 포맷에 맞게 저장 결과 + 전체 내용 출력.

---

## STEP 5 — 테스트 결과 기록 (사용자 보고 시)

사용자가 테스트 결과를 보고하면 기존 시나리오 파일을 업데이트한다.

### 트리거 조건

아래와 같은 사용자 발화 시 이 단계를 실행:
- "10개 중 9개 패스"
- "TC-003 실패"
- "테스트 결과 기록해줘"
- "전부 통과했어"
- "테스트 끝났어"
- 특정 TC에 대한 오류/문제 보고

### 실행 순서

1. `test-scenarios/` 디렉토리에서 가장 최근 시나리오 파일을 Read
   - 사용자가 파일명을 지정하면 해당 파일 사용
   - 지정하지 않으면 오늘 날짜 또는 가장 최근 파일 사용
2. 사용자 보고 내용을 파싱:
   - 전체 통과 → 모든 TC에 Pass 기록
   - 부분 통과 → 통과 TC는 Pass, 실패 TC는 Fail + 발견 문제 기록
   - 정보가 부족하면 한 번만 질문: "어떤 TC가 실패했고, 어떤 문제였나요?"
3. 시나리오 파일 내 해당 TC의 결과 섹션을 Edit으로 업데이트:
   ```
   **결과**: [x] Pass  또는  [x] Fail
   **테스트 일시**: YYYY-MM-DD HH:MM
   **발견 문제**: (Fail인 경우) 문제 설명
   **조치 상태**: ⬜ 미조치
   ```
4. 파일 하단의 `테스트 실행 요약` 테이블 업데이트
5. 채팅에 결과 요약 출력

### 조치 상태 업데이트

사용자가 수정 요청 → 수정 완료 흐름을 거칠 때 조치 상태도 함께 업데이트:

| 상태 | 의미 |
|------|------|
| ⬜ 미조치 | 문제 발견, 아직 수정 안 함 |
| 🔧 수정 중 | 수정 요청됨, 작업 진행 중 |
| ✅ 수정 완료 | 수정 후 재테스트 통과 |
| 🔁 재테스트 필요 | 수정했으나 재확인 필요 |

### 채팅 출력 포맷

```
📋 테스트 결과 기록 완료
📁 파일: test-scenarios/YYYYMMDD_기능명.md

| ID | 결과 | 발견 문제 | 조치 상태 |
|----|------|----------|----------|
| TC-001 | ✅ Pass | — | — |
| TC-007 | ❌ Fail | A 문제 설명 | ⬜ 미조치 |

총 N건: ✅ Pass N건 / ❌ Fail N건
```

---

## 실패 처리

| 실패 유형 | 처리 방법 |
|----------|----------|
| 기능 설명 부족 | STEP 1 질문 1회 실행 |
| 폴더 생성 실패 | Bash 재시도 → 실패 시 채팅에만 출력 |
| 파일 저장 실패 | 채팅에만 출력 후 저장 실패 안내 |
| references 로드 실패 | 경로 재확인 후 사용자에게 알림 |
| 시나리오 파일 없음 | "먼저 테스트 시나리오를 생성해주세요" 안내 |
| TC 번호 불일치 | 파일 내 TC 목록 보여주고 재확인 요청 |
