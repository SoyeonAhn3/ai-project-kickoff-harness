# kickoff-checklist Skill

project_kickoff.md의 프로필/아키텍처 내용을 분석하여 개발 착수 전 확인해야 할 항목을 자동 생성하고, 섹션 6으로 추가하는 스킬. 이 체크리스트를 모두 통과하면 개발을 시작할 수 있는 상태.

---

## 사전 조건

- 킥오프 문서(`{프로젝트명}_kickoff.md`)가 존재해야 함
- 대화 컨텍스트에 경로가 없으면 Glob으로 `**/*_kickoff.md`를 검색
  - 0개 발견: `/kickoff-start`를 먼저 실행하도록 안내
  - 1개 발견: 해당 파일 Read → YAML frontmatter에서 `last_skill` 확인
  - 2개 이상 발견: frontmatter에 `last_skill` 필드가 있는 파일만 대상, 여러 개면 사용자에게 선택 요청
- YAML frontmatter `last_skill`이 `kickoff-gap` 이상이어야 함
  - 아니면 `/kickoff-gap`을 먼저 실행하도록 안내

킥오프 문서(`{프로젝트명}_kickoff.md`)를 Read하여 아래 정보를 추출한다:

## STEP 2 — 체크리스트 생성

추출된 정보를 기반으로 체크리스트를 동적으로 생성한다.

### 체크리스트 구성 영역

**1. 환경 구성**
- 필요한 런타임/언어 설치 확인
- 필요한 패키지/라이브러리 설치 가능 확인
- 개발 환경 제약사항 충족 확인

**2. 데이터 & 접근**
- 데이터 소스 접근 가능 확인
- API 키/인증 정보 확보
- 샘플 데이터 준비 여부

**3. 외부 서비스**
- 필요한 외부 계정 생성/인증
- 외부 API 사용 한도 확인
- 네트워크/방화벽 접근 확인
- **(조건부) 배포 환경 사전 검증**: 섹션 5(gap 점검)에서 "배포 환경 접근 제한 리스크"가 지적된 외부 서비스가 있으면 아래 항목을 추가:
  - `배포 환경(클라우드)에서 해당 서비스 접근 PoC 수행` — 로컬이 아닌 실제 배포 환경에서 데이터 수집이 동작하는지 사전 검증
  - `대체 데이터 소스(Fallback) 확보` — 비공식 API/스크래핑이 차단될 경우 사용할 공식 API 또는 대체 서비스 확보

**4. 설계 확정**
- MVP 범위 최종 확정
- 핵심 데이터 구조 확정 (설계 인터뷰 진행한 경우)
- 미해결 의사결정 사항 없음 확인

**5. 프로젝트 설정**
- Git 저장소 초기화
- 프로젝트 폴더 구조 생성
- 기본 스킬 세팅 완료 (kickoff-skills)

### 목적별 추가 항목 (컨텍스트 기반)

킥오프 문서 `### 프로젝트 배경`(사용자 컨텍스트)이 존재하면 목적에 따라 추가 항목을 생성한다:

| 목적 | 추가 체크 항목 |
|---|---|
| 포트폴리오 | README 초안 작성, 데모 시나리오 구상, 스크린샷/GIF 촬영 계획 |
| 업무 자동화 | 기존 프로세스 문서화, AS-IS/TO-BE 정리 |
| 실무 투입 | 배포 환경 확인, 모니터링 도구 선정, 롤백 절차 수립 |
| 웹/앱 + 외부 사용자 | 인증 방식 확정, SSL 인증서 준비, 보안 점검 항목 확인 |

- 컨텍스트 없으면 공통 5개 영역만 생성
- 추가 항목은 "6. 목적별 확인" 영역으로 별도 그룹핑

### v2 설계 기반 산출물 (조건부)

v2 설계 스킬(섹션 9~12)이 작성된 경우, 체크리스트에 추가로 프로젝트 구조와 환경 설정을 생성한다. **v2 미수행 시 이 부분은 건너뜀.**

**7. 프로젝트 구조 (Project Structure)**

섹션 10-1(시스템 구조)의 컴포넌트를 파일/폴더로 매핑하여 트리 구조를 생성한다:
- 섹션 10-1 컴포넌트 → 폴더/파일 매핑
- 기술 스택(섹션 10-2) 관례에 맞는 구조 사용
- 테스트 디렉토리 포함
- 각 파일/폴더에 역할 주석 (섹션 10-1 기반)

**8. 환경 설정 (Configuration)**

섹션 10-2(기술 선택) + 외부 서비스 의존성에서 필요한 환경 변수를 추출한다:
- API 키, 접속 정보, 인증 토큰 등
- 각 변수의 출처 섹션 명시
- `.env.example` 파일 내용 생성

## 실패 처리

| 실패 유형 | 처리 방법 |
|---|---|
| 킥오프 문서 없음 | `/kickoff-start`를 먼저 실행하도록 안내 |
| 섹션 6 플레이스홀더 없음 | 파일 하단에 섹션 6을 새로 추가 |
| 프로필 내용이 너무 부족 | 최소한의 공통 체크리스트만 생성 + 안내 |
| 사용자가 항목 추가/삭제 요청 | 반영 후 재저장 |

---

## 주의사항

- 체크리스트는 "확인" 항목이지 "작업" 항목이 아님 — "구현하세요"가 아니라 "확인하세요"
- 프로젝트 내용에 없는 것을 추측해서 넣지 않음
- 설계 패스 시 "설계 확정" 영역은 최소화 (MVP 확정 정도만)
- 반드시 Read 먼저 → Edit으로 추가
