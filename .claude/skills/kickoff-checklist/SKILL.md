# kickoff-checklist Skill

project_kickoff.md의 프로필/아키텍처 내용을 분석하여 개발 착수 전 확인해야 할 항목을 자동 생성하고, 섹션 6으로 추가하는 스킬. 이 체크리스트를 모두 통과하면 개발을 시작할 수 있는 상태.

---

## 사전 조건

- `pre-requirement/project_kickoff.md` 파일이 존재해야 함
- 섹션 5(누락/모순 점검)까지 완료된 상태 권장
- 없으면 `/kickoff-gap`을 먼저 실행하도록 안내

---

## STEP 1 — 문서 읽기

`pre-requirement/project_kickoff.md`를 Read하여 아래 정보를 추출한다:

| 추출 대상 | 체크리스트 생성 항목 |
|---|---|
| 기술 스택 | 설치/환경 구성 확인 |
| 데이터 소스 | 접근 권한, API 키, 샘플 데이터 |
| 외부 서비스 | 계정 생성, 인증 설정 |
| MVP 기능 | 각 기능의 착수 가능 여부 |
| 제약사항 | 환경 제약 충족 여부 |
| 설계 구조 (있는 경우) | 스키마/설정 파일 확정 |

---

## STEP 2 — 체크리스트 생성

추출된 정보를 기반으로 체크리스트를 동적으로 생성한다.

### 체크리스트 구성 영역

**1. 환경 구성**
- 필요한 런타임/언어 설치 확인
- 필요한 패키지/라이브러리 설치 가능 확인
- 개발 환경 제약사항 충족 확인

**2. 데이터 & 접근**
- 데이터 소스 접근 가능 확인
- API 키/인증 정보 확보
- 샘플 데이터 준비 여부

**3. 외부 서비스**
- 필요한 외부 계정 생성/인증
- 외부 API 사용 한도 확인
- 네트워크/방화벽 접근 확인

**4. 설계 확정**
- MVP 범위 최종 확정
- 핵심 데이터 구조 확정 (설계 인터뷰 진행한 경우)
- 미해결 의사결정 사항 없음 확인

**5. 프로젝트 설정**
- Git 저장소 초기화
- 프로젝트 폴더 구조 생성
- 기본 스킬 세팅 완료 (kickoff-skills)

## 실패 처리

| 실패 유형 | 처리 방법 |
|---|---|
| project_kickoff.md 없음 | `/kickoff-profile`을 먼저 실행하도록 안내 |
| 섹션 6 플레이스홀더 없음 | 파일 하단에 섹션 6을 새로 추가 |
| 프로필 내용이 너무 부족 | 최소한의 공통 체크리스트만 생성 + 안내 |
| 사용자가 항목 추가/삭제 요청 | 반영 후 재저장 |

---

## 주의사항

- 체크리스트는 "확인" 항목이지 "작업" 항목이 아님 — "구현하세요"가 아니라 "확인하세요"
- 프로젝트 내용에 없는 것을 추측해서 넣지 않음
- 설계 패스 시 "설계 확정" 영역은 최소화 (MVP 확정 정도만)
- 반드시 Read 먼저 → Edit으로 추가
