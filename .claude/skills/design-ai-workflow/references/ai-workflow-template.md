# AI Workflow Template

`design-ai-workflow` 스킬이 섹션 12(AI 워크플로우)를 `_kickoff.md`에 작성할 때 참조하는 구조 가이드.
**이 스킬은 AI 프로젝트에서만 활성화된다.**

---

## 활성화 조건

| 필드 | 위치 | 판정 |
|---|---|---|
| 프로젝트 유형 | 섹션 1 → 기본 정보 → 유형 | "AI" 포함 시 활성 |

- **AI 프로젝트**: 섹션 12 생성 진행
- **비AI 프로젝트**: 자동 스킵 → `last_skill`을 `design-ai-workflow`로 갱신 후 `/kickoff-gap` 안내
- 스킵 시 출력: `"AI 프로젝트가 아니므로 섹션 12(AI 워크플로우)를 건너뜁니다. → /kickoff-gap → /kickoff-checklist"`

## AI 유형별 워크플로우 패턴

### LLM 기반 (프롬프트 엔지니어링)

```
입력 → 입력 검증 → 프롬프트 조립 → LLM API 호출 → 출력 파싱 → 출력 검증 → 후처리 → 응답
                                          ↓ (실패 시)
                                     폴백 처리
```

**핵심 설계 항목**:
- 프롬프트 템플릿 관리 (버전 관리, A/B 테스트)
- 출력 파싱 (JSON 모드, 구조화된 출력)
- 토큰 최적화 (캐싱, 요약, 청킹)
- 프롬프트 인젝션 방어

### RAG (Retrieval-Augmented Generation)

```
질의 → 질의 변환 → 벡터 검색 → 컨텍스트 선택 → 프롬프트 조립 → LLM → 응답
                        ↓
                   인덱스 갱신 파이프라인
```

**핵심 설계 항목**:
- 임베딩 모델 선택
- 벡터 DB 선택 (Pinecone, Chroma, Weaviate 등)
- 청킹 전략 (크기, 오버랩)
- 검색 품질 평가 (recall, precision)

### 커스텀 모델 학습

```
데이터 수집 → 전처리 → 학습 → 평가 → 모델 레지스트리 → 서빙 API
                                              ↓
                                         모니터링 → 재학습 트리거
```

**핵심 설계 항목**:
- 학습 데이터 파이프라인
- 실험 추적 (MLflow, W&B 등)
- 모델 버저닝 및 배포
- 데이터/모델 드리프트 감지

### Agent / Tool-use

```
입력 → 의도 파악 → 계획 수립 → 도구 선택 → 도구 실행 → 결과 통합 → 응답
                       ↑                        ↓
                       └──── 반복 (필요 시) ────┘
```

**핵심 설계 항목**:
- 도구 정의 (이름, 설명, 파라미터)
- 실행 루프 제한 (최대 반복, 타임아웃)
- 도구 실행 권한/안전성
- 중간 결과 로깅

## 프롬프트 설계 원칙

1. **역할 명시**: 시스템 프롬프트에 AI의 역할과 제약 명시
2. **구조화된 출력**: JSON 등 파싱 가능한 형식 요청 (가능한 경우)
3. **예시 포함**: Few-shot 예시로 출력 품질 향상
4. **변수 분리**: 하드코딩 대신 템플릿 변수 사용
5. **토큰 효율**: 불필요한 지시 제거, 핵심만 포함
6. **버전 관리**: 프롬프트를 코드처럼 버전 관리

## 비용 추정 가이드

| 항목 | 추정 방법 |
|---|---|
| 건당 비용 | (입력 토큰 × 입력 단가) + (출력 토큰 × 출력 단가) |
| 일일 비용 | 건당 비용 × 예상 일일 호출 수 |
| 월간 비용 | 일일 비용 × 30 |
| 비용 최적화 | 캐싱, 경량 모델 혼용, 배치 처리 |

---

## 폴백 설계 원칙

1. **단계적 폴백**: 재시도 → 대체 모델 → 기본값 → 에러 메시지
2. **사용자 투명성**: 폴백 발생 시 사용자에게 알림 (품질 저하 가능성)
3. **비용 제어**: 재시도 횟수 제한, 비용 한도 설정
4. **로깅**: 모든 폴백 이벤트를 기록하여 패턴 파악
