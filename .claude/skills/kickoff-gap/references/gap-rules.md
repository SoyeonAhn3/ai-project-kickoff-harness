# 누락/모순 점검 규칙

## 점검 원칙

1. **명확한 모순만** 잡는다 — 강한 확신이 있는 것만 지적
2. **애매한 건** 사용자에게 질문 — AI가 임의로 판단하지 않음
3. **오탐 최소화** — 없는 문제를 만들어내는 것보다 놓치는 게 낫다
4. **킥오프 수준** — 구현 레벨 세부사항은 점검하지 않음

---

## 점검 카테고리

### 카테고리 1 — 기술 선택 불일치

프로필에 명시된 기술 스택이 서로 충돌하거나, 제약사항과 맞지 않는 경우.

**점검 항목:**
- 기술 스택과 제약사항 간 충돌 (예: "Docker 불가"인데 컨테이너 기반 아키텍처)
- 프론트엔드/백엔드 기술 간 호환성 문제
- 선택한 도구가 데이터 볼륨을 감당 불가 (예: SQLite에 초당 1만 건)

**판정 기준:**
- 명확한 기술적 비호환 → **모순**으로 지적
- "보통 이 조합은 잘 안 씀" 수준 → 지적하지 않음

---

### 카테고리 2 — 데이터 흐름 누락

파이프라인 흐름에서 빠진 단계가 있는 경우.

**점검 항목:**
- 데이터 소스는 정의되었지만 수집 방법이 없음
- 처리 결과는 있지만 저장 위치가 없음
- 입력 데이터의 형식 변환이 필요한데 명시되지 않음
- 외부 서비스 호출은 있지만 인증/키 관리 언급 없음

**판정 기준:**
- 흐름 상 반드시 필요한 단계 누락 → **누락**으로 지적
- "있으면 좋은" 수준 → 지적하지 않음

---

### 카테고리 3 — 시간/스케줄 충돌

실행 주기, 갱신 시간, 의존 관계 사이에 타이밍 문제가 있는 경우.

**점검 항목:**
- 스케줄러 실행 시간과 데이터 갱신 시간 충돌 (예: 데이터가 09시에 갱신되는데 배치가 08시 실행)
- 의존 데이터가 준비되기 전에 후속 처리가 시작
- 실시간 요구사항인데 배치 처리 설계

**판정 기준:**
- 명시적 시간 충돌 → **모순**으로 지적
- 시간이 명시되지 않은 경우 → **애매** → 사용자에게 질문

---

### 카테고리 4 — 실패 처리 누락

시스템이 실패할 수 있는 상황에 대한 대응이 정의되지 않은 경우.

**점검 항목:**
- AI 결과가 틀렸을 때의 폴백이 없음
- 외부 서비스 장애 시 동작이 정의되지 않음
- 데이터 중복 방지 방식 누락
- 알림/모니터링 대상이 정의되지 않음

**판정 기준:**
- 핵심 흐름에서 실패 시 시스템이 멈추는 경우 → **누락**으로 지적
- 부가 기능의 실패 처리 → 지적하지 않음 (킥오프 수준)

---

### 카테고리 5 — MVP 범위 모순

MVP에 포함된 기능과 다른 항목 간 불일치.

**점검 항목:**
- MVP에 포함된 기능이 "이후 확장"으로 분류된 의존성을 필요로 함
- MVP 범위가 제약사항(기한, 인력)에 비해 과다
- 핵심 기능이 MVP에서 빠져있음

**판정 기준:**
- 기능 의존성 충돌 → **모순**으로 지적
- 범위 과다 여부 → **애매** → 사용자에게 "이 범위가 [기한] 내에 가능한가요?" 질문

---

### 카테고리 6 — 평가 지적 항목 재확인

evaluate 스킬에서 🟡/🔴로 지적된 항목이 킥오프 문서 내에서 일관되게 반영되었는지 확인.

**점검 항목:**
- evaluate에서 범위 조정이 이루어진 경우, 킥오프 문서의 해당 섹션이 수정되었는지
- 조정 결과와 기존 섹션(기술 스택, MVP 범위 등) 간 새로운 모순이 없는지
- evaluate를 실행하지 않은 경우 이 카테고리는 건너뜀

**판정 기준:**
- 조정 내용과 기존 섹션 간 충돌 → **모순**으로 지적
- evaluate 미실행 → 점검 불가 (건너뜀)

---

## 설계 인터뷰 패스 시 점검 범위

설계 인터뷰를 패스한 경우 (섹션 2~4 없음):
- 카테고리 2, 3 → 점검 불가 (데이터 없음)
- 카테고리 1 → 제약사항과 기술 스택 간 충돌만 점검
- 카테고리 4 → 프로필에서 언급된 범위만 점검
- 카테고리 5 → 정상 점검

---

## 결과 기록 형식

### 모순/누락 발견 시

```markdown
| # | 유형 | 내용 | 상태 |
|---|---|---|---|
| 1 | 모순 | [구체적 내용] | 🔴 미해결 / ✅ 해결 |
| 2 | 누락 | [구체적 내용] | 🔴 미해결 / ✅ 해결 |
```

### 해결된 항목 기록

```markdown
| # | 유형 | 내용 | 상태 | 해결 방법 |
|---|---|---|---|---|
| 1 | 모순 | [내용] | ✅ 해결 | [사용자 결정 내용] |
```

### 점검 결과 없음

```markdown
점검 결과: 명확한 누락/모순이 발견되지 않았습니다.
```
