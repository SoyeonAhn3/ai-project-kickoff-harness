# 프로젝트 유형 분류 기준

## 7개 유형 정의

| # | 유형 | 설명 | 대표 키워드 |
|---|---|---|---|
| 1 | AI 프로젝트 | 머신러닝/딥러닝 모델 학습·추론·서빙이 핵심 | AI, 모델, 예측, 분류, 감지, NLP, LLM, 추천, 이상탐지, 학습 |
| 2 | 자동화 프로젝트 | 반복 업무를 스크립트/봇으로 대체 | 자동화, 봇, 스케줄, 크론, RPA, 매크로, 반복, 알림 |
| 3 | BI 대시보드 | 데이터 시각화·리포팅이 핵심 | 대시보드, 시각화, 리포트, KPI, 차트, 모니터링, Grafana, PowerBI |
| 4 | 데이터 파이프라인 | 데이터 수집·변환·적재 흐름 구축 | ETL, 파이프라인, 수집, 적재, 변환, 배치, 스트리밍, Airflow |
| 5 | 웹앱 프로젝트 | 사용자 대면 웹 서비스 구축 | 웹, 프론트엔드, 백엔드, API, 로그인, 페이지, SaaS |
| 6 | CLI 도구 | 터미널에서 실행하는 명령줄 도구 | CLI, 터미널, 명령어, 스크립트, 도구, 유틸리티 |
| 7 | 라이브러리 | 다른 프로젝트에서 import해서 쓰는 모듈 | 라이브러리, SDK, 패키지, 모듈, pip, npm |

## 판별 규칙

### 1차: 키워드 매칭

사용자 입력에서 위 키워드와 가장 많이 일치하는 유형을 후보로 선정한다.

### 2차: 맥락 분석

키워드만으로 부족할 때 아래 질문으로 판별:
- "최종 산출물이 무엇인가?" → 모델이면 AI, 화면이면 웹앱/BI, 파일이면 파이프라인
- "누가 사용하는가?" → 사람이면 웹앱/BI/CLI, 시스템이면 파이프라인/라이브러리
- "주기적으로 돌아가는가?" → 예 → 자동화/파이프라인

### 3차: 복합 유형 처리

- 2개 이상 유형에 걸치면 "주요 유형 + 부가 유형" 형태로 표기
- 예: "데이터 파이프라인 + AI", "웹앱 + BI 대시보드"
- 주요 유형 = 프로젝트의 핵심 목적에 해당하는 유형
- 부가 유형 = 핵심을 지원하는 보조 유형

### 판별 실패 시

아래 조건에 해당하면 사용자에게 직접 선택 요청:
- 키워드 일치가 2개 유형에서 동률
- 입력이 너무 짧아 맥락 파악 불가 (5단어 미만)
- 어떤 유형에도 해당하지 않는 경우

선택지 제시 형식:
```
아래 중 가장 가까운 유형을 선택해주세요:
1. AI 프로젝트
2. 자동화 프로젝트
3. BI 대시보드
4. 데이터 파이프라인
5. 웹앱 프로젝트
6. CLI 도구
7. 라이브러리
8. 기타 (직접 입력)
```
