# kickoff-start Skill

사용자의 프로젝트 아이디어를 자유 텍스트로 입력받고, 프로젝트 유형을 판별하여 확인받는 킥오프 진입점 스킬.

---

## 사전 조건

- `kickoff-context` 완료 권장 (미완료 시에도 정상 동작)
- context 데이터가 대화 컨텍스트에 있으면 상태 파일 섹션 0-1로 포함

---

## STEP 1 — 아이디어 입력 요청

사용자에게 프로젝트 아이디어를 자유롭게 설명하도록 요청한다.

출력:

```
## 🚀 프로젝트 킥오프를 시작합니다

어떤 프로젝트를 만들고 싶으신가요?
자유롭게 설명해주세요. (한국어/영어 모두 가능)

예시:
- "생산 데이터를 보고 이상을 감지하고 AI가 원인을 분석해주는 시스템"
- "매일 이메일로 오는 보고서를 자동으로 정리해서 팀 채널에 올려주는 봇"
- "사내 데이터를 실시간으로 모니터링하는 대시보드"
```

### 입력 검증

- 입력이 5단어 미만이면 추가 설명 요청:

```
조금 더 구체적으로 설명해주시면 더 정확한 기획이 가능합니다.
- 어떤 문제를 해결하고 싶은지
- 누가 사용할 건지
- 어떤 결과물을 기대하는지
하나라도 추가해주세요.
```

---

## STEP 2 — 프로젝트 유형 판별

`references/project-types.md`의 분류 기준을 참조하여 유형을 판별한다.

### 판별 순서

1. 사용자 입력에서 키워드 매칭 → 후보 유형 선정
2. 키워드만으로 부족하면 맥락 분석 (산출물, 사용자, 주기성)
3. 2개 이상 유형에 걸치면 복합 유형으로 판별 (주요 + 부가)
4. 판별 불가 시 사용자에게 직접 선택 요청

---

## STEP 3 — 판별 결과 확인

판별된 유형을 사용자에게 제시하고 확인을 받는다.

### 단일 유형인 경우

```
## 📋 프로젝트 유형 판별 결과

**유형**: [유형명]
**판별 근거**: [입력에서 어떤 부분이 이 유형에 해당하는지 1줄 설명]

이 유형이 맞나요? (맞으면 "맞아" / 아니면 원하는 유형을 말씀해주세요)
```

### 복합 유형인 경우

```
## 📋 프로젝트 유형 판별 결과

**주요 유형**: [유형명] — [핵심 목적 한 줄]
**부가 유형**: [유형명] — [보조 역할 한 줄]
**판별 근거**: [1줄 설명]

이 유형 조합이 맞나요? (맞으면 "맞아" / 아니면 수정해주세요)
```

### 판별 불가인 경우

`references/project-types.md`의 선택지 목록을 제시하고 사용자가 직접 선택하도록 한다.

---

## STEP 4 — 프로젝트명 및 산출물 경로 설정

유형 확정 후, 프로젝트명과 산출물 저장 경로를 확인한다.

### 출력 형식

```
## 📁 프로젝트 설정

**1. 프로젝트 이름**을 지정해주세요.
   미입력 시 자동 생성합니다.

**2. 산출물 저장 경로**를 지정해주세요.
   미입력 시: `output/{프로젝트명}/`
```

### 처리 규칙

- **프로젝트명 미입력**: 아이디어에서 간결한 영문 snake_case 이름을 자동 생성 (예: `stock_macro_analyzer`)
- **경로 미입력**: `output/{프로젝트명}/` 을 기본 경로로 사용
- **경로 지정 시**: 사용자가 지정한 경로 그대로 사용
- 프로젝트명은 이후 파일명에 사용됨: `{프로젝트명}_kickoff.md`, `kickoff_state.md`

---

## STEP 5 — 확정 및 상태 저장

프로젝트명과 경로가 확정되면 상태 파일을 생성한다.

### 동작

1. 산출물 경로 폴더 존재 확인 (없으면 생성)
2. `{산출물 경로}/kickoff_state.md` 파일 생성 (Write)
3. 다음 단계 안내

### 상태 파일 작성 형식

```markdown
# Kickoff 진행 상태

마지막 완료: kickoff-start
마지막 갱신: [현재 시각 ISO 8601]

## 0. 산출물 설정

- 프로젝트명: [확정된 프로젝트명]
- 산출물 경로: [확정된 경로]
- 킥오프 문서: {산출물 경로}/{프로젝트명}_kickoff.md

## 0-1. 사용자 컨텍스트 (kickoff-context)

> ⚠️ kickoff-context를 실행하지 않은 경우 이 섹션은 생략됩니다.

| 항목 | 내용 |
|---|---|
| 프로젝트 목적 | [포트폴리오/업무 자동화/학습/실무 투입] |
| 대상 독자/평가자 | [채용 담당자/팀장/교수/본인] |
| 현재 기술 수준 | [초급/중급/고급] + [구체 역량] |
| 목표 직무 | [직무명 또는 "없음"] |
| 시간 예산 | [기간] |
| 차별화 방향 | [키워드 또는 "없음"] |

## 1. 프로젝트 정보 (kickoff-start)

- 주 유형: [유형명]
- 부가 유형: [유형명 또는 "없음"]
- 원본 아이디어: [사용자 입력 전문]
```

### 안내 출력

```
## ✅ 프로젝트 킥오프 시작

**프로젝트명**: [프로젝트명]
**유형**: [확정된 유형]
**산출물 경로**: [경로]
**킥오프 문서**: [경로]/[프로젝트명]_kickoff.md

진행 상태가 `[경로]/kickoff_state.md`에 저장되었습니다.
다음 단계로 `/kickoff-interview`를 실행하면 기획 인터뷰가 시작됩니다.
```

---

## 출력 형식

이 스킬은 **산출물 폴더와 상태 파일을 생성**한다:
- 폴더: `{산출물 경로}/` (없으면 생성)
- 파일: `{산출물 경로}/kickoff_state.md`
- 이후 모든 스킬이 이 파일을 읽고 갱신한다.

---

## 실패 처리

| 실패 유형 | 처리 방법 |
|---|---|
| 입력이 너무 짧음 (5단어 미만) | 추가 설명 요청 (STEP 1 참고) |
| 유형 판별 불가 | 7개 유형 + 기타 선택지 제시 |
| 사용자가 판별 결과에 동의하지 않음 | 사용자가 말한 유형으로 재설정 |
| 사용자가 "기타"를 선택 | 사용자 입력을 그대로 유형명으로 사용 |

---

## 주의사항

- 유형 판별의 정확도보다 사용자 확인이 우선. 틀려도 사용자가 수정 가능.
- 복합 유형 시 반드시 주요/부가를 구분.
- 이 스킬은 파일을 생성하지 않음. 대화 컨텍스트 전달만 담당.
- 사용자가 이미 유형을 알고 있으면 ("AI 프로젝트 만들 거야") 판별 과정 생략하고 바로 확인만 받는다.
