# kickoff-start Skill

사용자의 프로젝트 아이디어를 자유 텍스트로 입력받고, 프로젝트 유형을 판별하여 확인받는 킥오프 진입점 스킬.

---

## 사전 조건

- `kickoff-context` 완료 권장 (미완료 시에도 정상 동작)
- context 데이터가 대화 컨텍스트에 있으면 킥오프 문서 Section 1 `### 프로젝트 배경`으로 포함

---

## STEP 1 — 언어 선택 및 아이디어 입력

사용자에게 먼저 언어를 선택받고, 이후 프로젝트 아이디어를 자유롭게 설명하도록 요청한다.

### 언어 선택 규칙

- 사용자가 번호/언어명으로 선택하면 해당 언어로 진행
- 별도 선택 없이 바로 아이디어를 설명하면, 사용한 언어를 자동 감지하여 설정
- 선택된 언어는 YAML frontmatter `language` 필드에 저장 (`ko` 또는 `en`)
- **이후 모든 스킬의 질문, 출력, 섹션 내용을 선택된 언어로 작성**

출력:

```
## 🚀 프로젝트 킥오프 / Project Kickoff

**언어를 선택해주세요 / Please select a language:**
1. 한국어
2. English

(이후 모든 산출물이 선택한 언어로 작성됩니다 / All outputs will be in the selected language)
```

### 입력 검증

- 입력이 5단어 미만이면 추가 설명 요청:

```
조금 더 구체적으로 설명해주시면 더 정확한 기획이 가능합니다.
- 어떤 문제를 해결하고 싶은지
- 누가 사용할 건지
- 어떤 결과물을 기대하는지
하나라도 추가해주세요.
```

---

## STEP 2 — 프로젝트 유형 판별

`references/project-types.md`의 분류 기준을 참조하여 유형을 판별한다.

### 판별 순서

1. 사용자 입력에서 키워드 매칭 → 후보 유형 선정
2. 키워드만으로 부족하면 맥락 분석 (산출물, 사용자, 주기성)
3. 2개 이상 유형에 걸치면 복합 유형으로 판별 (주요 + 부가)
4. 판별 불가 시 사용자에게 직접 선택 요청

---

## STEP 3 — 판별 결과 확인

판별된 유형을 사용자에게 제시하고 확인을 받는다.

### 단일 유형인 경우

```
## 📋 프로젝트 유형 판별 결과

**유형**: [유형명]
**판별 근거**: [입력에서 어떤 부분이 이 유형에 해당하는지 1줄 설명]

이 유형이 맞나요? (맞으면 "맞아" / 아니면 원하는 유형을 말씀해주세요)
```

### 복합 유형인 경우

```
## 📋 프로젝트 유형 판별 결과

**주요 유형**: [유형명] — [핵심 목적 한 줄]
**부가 유형**: [유형명] — [보조 역할 한 줄]
**판별 근거**: [1줄 설명]

이 유형 조합이 맞나요? (맞으면 "맞아" / 아니면 수정해주세요)
```

### 판별 불가인 경우

`references/project-types.md`의 선택지 목록을 제시하고 사용자가 직접 선택하도록 한다.

---

## STEP 4 — 프로젝트명 및 산출물 경로 설정

유형 확정 후, 프로젝트명과 산출물 저장 경로를 확인한다.

### 출력 형식

```
## 📁 프로젝트 설정

**1. 프로젝트 이름**을 지정해주세요.
   미입력 시 자동 생성합니다.

**2. 산출물 저장 경로**를 지정해주세요.
   미입력 시: `../{프로젝트명}/pre-requirement/`
```

### 처리 규칙

- **프로젝트명 미입력**: 아이디어에서 간결한 영문 snake_case 이름을 자동 생성 (예: `stock_macro_analyzer`)
- **경로 미입력**: `../{프로젝트명}/pre-requirement/` 을 기본 경로로 사용 (하네스 상위 디렉토리)
- **경로 지정 시**: 사용자가 지정한 경로 그대로 사용
- 프로젝트명은 이후 파일명에 사용됨: `{프로젝트명}_kickoff.md`

---

## STEP 5 — 프로젝트 폴더 생성 및 킥오프 문서 작성

프로젝트명과 경로가 확정되면 프로젝트 폴더와 킥오프 문서를 생성한다.

### 동작

1. 하네스 상위 디렉토리에 프로젝트 폴더 생성: `../{프로젝트명}/`
2. 하위 폴더 생성: `../{프로젝트명}/pre-requirement/`
3. 킥오프 문서 생성: `../{프로젝트명}/pre-requirement/{프로젝트명}_kickoff.md` (Write)
4. 다음 단계 안내

### 폴더가 이미 존재하는 경우

```
{프로젝트명}/ 폴더가 이미 존재합니다.
1. 그대로 사용 (기존 내용 유지)
2. 다른 이름 지정
3. 취소
```

### 킥오프 문서 작성 형식

`references/profile-template.md`를 참조하여 아래 구조로 생성한다.

## 출력 형식

이 스킬은 **프로젝트 폴더와 킥오프 문서를 생성**한다:
- 폴더: `../{프로젝트명}/pre-requirement/` (없으면 생성)
- 파일: `../{프로젝트명}/pre-requirement/{프로젝트명}_kickoff.md`
- 이후 모든 스킬이 이 파일을 읽고 해당 섹션을 채운다.

---

## 실패 처리

| 실패 유형 | 처리 방법 |
|---|---|
| 입력이 너무 짧음 (5단어 미만) | 추가 설명 요청 (STEP 1 참고) |
| 유형 판별 불가 | 7개 유형 + 기타 선택지 제시 |
| 사용자가 판별 결과에 동의하지 않음 | 사용자가 말한 유형으로 재설정 |
| 사용자가 "기타"를 선택 | 사용자 입력을 그대로 유형명으로 사용 |
| 프로젝트 폴더 생성 실패 | 에러 메시지 출력 + 수동 생성 안내 |
| 기존 킥오프 문서 존재 | "덮어쓰기 / 다른 이름 / 취소" 선택지 제시 |

---

## 주의사항

- 유형 판별의 정확도보다 사용자 확인이 우선. 틀려도 사용자가 수정 가능.
- 복합 유형 시 반드시 주요/부가를 구분.
- 이 스킬은 프로젝트 폴더와 킥오프 문서를 생성한다. 이후 모든 스킬이 이 문서를 읽고 해당 섹션을 채운다.
- 사용자가 이미 유형을 알고 있으면 ("AI 프로젝트 만들 거야") 판별 과정 생략하고 바로 확인만 받는다.
- 기존 폴더가 있을 때 무조건 덮어쓰지 않음 — 사용자 확인 필수
